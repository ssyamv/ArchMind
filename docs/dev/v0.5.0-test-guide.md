## 一、#62 RBAC 精细权限系统

### 1.1 角色迁移验证

```sql
-- 验证旧 member 角色已迁移为 editor（不应有 member 角色存在）
SELECT role, COUNT(*) FROM workspace_members GROUP BY role;
-- 预期：结果中不应出现 role = 'member'
```

### 1.2 邀请成员时角色选择

**操作步骤**：
1. 进入工作区设置 → 成员管理
2. 点击「邀请成员」
3. 检查角色下拉菜单

**预期**：选项包含「编辑者 / 只读者 / 访客 / 管理员」，不再只有「成员」

### 1.3 权限边界测试

准备：准备 3 个测试账号，分别邀请为 `editor`、`viewer`、`guest`

| 操作 | editor | viewer | guest | 期望结果 |
|------|--------|--------|-------|---------|
| 上传文档 | ✅ | ❌ | ❌ | viewer/guest 返回 403 |
| 查看文档 | ✅ | ✅ | ❌ | guest 返回 403 |
| 创建 PRD | ✅ | ❌ | ❌ | viewer/guest 返回 403 |
| 删除 PRD | ❌ | ❌ | ❌ | 全部返回 403 |
| 修改工作区设置 | ❌ | ❌ | ❌ | 全部返回 403 |

**操作**：用 `editor` 账号尝试删除 PRD：

```bash
curl -X DELETE http://localhost:3000/api/v1/prd/<prd-id> \
  -H "Cookie: auth_token=<editor_token>"
# 预期返回 403，message 包含 "权限不足"
```

### 1.4 权限 API 验证

```bash
# 获取当前用户有效权限
GET /api/v1/workspaces/<workspace-id>/permissions

# 预期响应（editor 角色）：
{
  "success": true,
  "data": {
    "role": "editor",
    "permissions": ["document:read", "document:write", "document:delete", "prd:read", "prd:write", ...]
  }
}
```

```bash
# 获取工作区角色权限矩阵
GET /api/v1/workspaces/<workspace-id>/roles
# 预期：返回 owner/admin/editor/viewer/guest 5 个角色及各自权限
```

### 1.5 修改成员角色

```bash
PATCH /api/v1/workspaces/<workspace-id>/members/<user-id>/role
Body: { "role": "viewer" }
# 用 admin 账号执行 → 预期 200
# 用 editor 账号执行 → 预期 403
# 尝试设置 role = "owner" → 预期 400（schema 限制）
```

## 三、#66 逻辑图谱生成

### 3.1 从 PRD 生成逻辑图

**操作步骤**：
1. 进入一个已有内容的 PRD 详情页
2. 点击「生成逻辑图」按钮（新增）
3. 在弹出 Dialog 中选择图形类型（如「流程图」）
4. 点击确认，观察是否有流式输出

**预期**：跳转到 `/logic-maps/<id>` 页面，Mermaid 图形逐步渲染

**4 种类型均需验证**：`flowchart`、`sequence`、`state`、`class`

### 3.2 SSE 流式生成 API

```bash
POST /api/v1/logic-maps/generate
Content-Type: application/json
Accept: text/event-stream
{
  "prdId": "<prd-id>",
  "type": "flowchart",
  "title": "测试流程图"
}
# 预期：SSE 流式返回 Mermaid 代码片段
```

### 3.3 逻辑图 CRUD

```bash
# 列表
GET /api/v1/logic-maps?workspaceId=<workspace-id>

# 详情
GET /api/v1/logic-maps/<id>

# 更新（修改标题或 mermaid_code）
PATCH /api/v1/logic-maps/<id>
{ "title": "新标题", "mermaidCode": "flowchart TD\n  A-->B" }

# 删除
DELETE /api/v1/logic-maps/<id>
```

### 3.4 前端渲染验证

1. 访问 `/logic-maps/<id>`，检查图形是否正确渲染
2. 在代码编辑区输入错误的 Mermaid 语法（如 `flowchart TD\n  A-->`）
3. **预期**：显示代码 + 错误提示，页面不崩溃
4. 点击「导出 SVG」，验证下载的文件可正常打开

## 七、#68 文档智能分类与自动标签

### 7.1 上传文档后触发自动标签

**操作步骤**：
1. 上传一份 PDF/Markdown 文档
2. 等待文档向量化完成（status = completed）
3. 等待约 30 秒
4. 查看文档详情，检查是否出现「AI 推荐」角标

```sql
-- 直接查看数据库
SELECT id, title, auto_summary, auto_category, auto_tags, auto_confidence
FROM documents
WHERE id = '<doc-id>';
-- 预期：auto_tags、auto_category 有值，auto_confidence > 0
```

### 7.2 确认 AI 推荐标签

**UI 操作**：
1. 进入文档详情，置信度 > 0.7 的文档应显示「自动标签建议」横幅
2. 点击「一键确认」

```bash
POST /api/v1/documents/<doc-id>/confirm-tags
{ "workspaceId": "<workspace-id>" }
# 预期：auto_tags 写入 tags 表，auto_tags_confirmed = true
# 再次调用 GET /api/v1/documents/<doc-id>，tags 字段应包含 auto_tags 内容
```

### 7.3 置信度低于 0.7 时不显示推荐

```sql
-- 手动将某文档置信度设为低值
UPDATE documents SET auto_confidence = 0.5 WHERE id = '<doc-id>';
-- 刷新页面，该文档不应显示「AI 推荐」提示
```
